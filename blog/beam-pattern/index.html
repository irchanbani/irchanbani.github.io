Beam Pattern: Calling External Services Using Retrofit For Data Enrichment

Background

For the past two months, me and my team had deployed pipeline using Apache Beam in production environment. The pipeline step is very simple just filter data by event, enrich data by user id, then insert to BigQuery. We expect that every event that will flow to the pipeline will be stored in BigQuery immediately (real-time). As you can guess, what we had in mind are definitely different with what happen in reality. Watermark is delayed up to 20 hours, isn’t it insane?

Attempt

First suspect is pubsub attributes(timestamp attribute), because we only use attribute id. As watermark is a sign of completeness, so we think if we add timestamp attribute to pubsub then watermark will advanced and not delayed like that. Is it success? well the answer is no.

Second suspect is fixed windows after read pubsub. As we just use simple windows, we think that we need to customize trigger in fixed windows. After the customization of trigger and watermark as mentioned in beam documentation, the pipeline is still delayed. After reading beam documentation and try to debug the pipeline using DirectRunner, we found that we can remove fixed windows. The reason is because when pubsub message flow to the pipeline, it can use global windows that created in the pipeline.
